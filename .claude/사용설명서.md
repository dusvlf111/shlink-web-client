## 1. 서비스 개요
Shlink Web Client는 단축 URL 서버(Shlink)를 브라우저에서 관리하는 도구입니다.

이 앱에서 할 수 있는 일:
1. 여러 Shlink 서버를 등록하고 전환
2. 서버별 단축 URL 관리 화면 사용
3. 사용자별 설정 저장(테마/목록 동작 등)
4. UTM 링크 생성 및 템플릿/태그 관리
5. 관리자 권한일 때 사용자 승인/비활성화/권한 변경

---

## 2. 처음 시작하기
### 2-1. 준비물
1. Shlink 서버 URL
2. Shlink API Key
3. 로그인 계정(조직 환경에서 PocketBase 인증 사용 시)

### 2-2. 접속 후 첫 작업
1. 로그인
2. 서버 추가
3. 서버 화면 진입
4. 설정 확인
5. 필요하면 UTM 작업 시작

### 2-3. 비전공자 핵심 포인트
1. 가장 먼저 서버를 등록해야 대부분 기능이 열립니다.
2. 서버는 여러 개 등록 가능하며, 상단 Servers 메뉴에서 전환합니다.
3. 관리자 계정이 아니면 사용자 관리 메뉴는 보이지 않습니다.

## 3. 화면 구조 한눈에 보기
앱은 크게 아래 영역으로 구성됩니다.

1. 상단 헤더
- 홈 로고
- Settings
- Servers 드롭다운
- 사용자 관리(관리자만)
- 로그아웃

2. 본문 영역
- 홈/서버/설정/UTM/관리자 페이지 내용 표시

3. 하단 버전 영역
- 앱/서버 버전 정보 표시

4. 우하단 플로팅 버튼
- 일반 화면에서는 UTM 관리로 이동
- UTM 화면에서는 HOME으로 이동

---

## 4. 빠른 시작: 처음 10분 가이드
### Step 1. 로그인
이메일, 비밀번호로 로그인합니다.

### Step 2. 서버 등록
Add a server에서 이름, URL, API Key 입력 후 생성합니다.

### Step 3. 서버 선택
상단 Servers 드롭다운에서 방금 등록한 서버를 선택합니다.

### Step 4. 기본 설정 점검
Settings에서 테마/목록 설정 등을 확인합니다.

### Step 5. UTM 필요 시 바로 시작
우하단 UTM 관리 버튼을 눌러 UTM 빌더로 이동합니다.

## 5. 공통 UI 사용법
### 5-1. 상단 메뉴
1. Shlink 로고 클릭: 홈 이동
2. Settings: 환경설정 이동
3. Servers: 서버 전환/관리
4. 로그아웃: 현재 계정 세션 종료

### 5-2. 검색/목록 패턴
서버 목록 등 표 형태 화면은 검색창으로 즉시 필터링됩니다.

### 5-3. 저장/취소 패턴
폼 화면은 보통 Save/Create와 Cancel 버튼 조합입니다.
취소 시 이전 화면으로 돌아갑니다.

## 6. 페이지별 상세 사용법
## 6-A. 로그인/회원가입 페이지
### 주요 기능
1. 로그인
2. 회원가입 신청

### 회원가입 후 상태
1. 가입하면 바로 사용되지 않고 승인 대기 상태가 될 수 있습니다.
2. 관리자 승인 후 로그인 가능합니다.

### 자주 나오는 메시지
1. 승인 대기 계정 안내
2. 비활성화 계정 안내

---

## 6-B. 홈 페이지
### 주요 기능
1. 등록된 서버 목록 확인
2. 서버 선택 진입
3. 서버가 없으면 Add a server 바로 이동

### 자동 연결
auto-connect로 지정된 서버가 있으면 홈 진입 시 자동 연결됩니다.

---

## 6-C. 서버 추가 페이지
### 주요 기능
1. 단일 서버 추가
2. 서버 파일 Import
3. 중복 서버 감지 후 확인 모달 제공

### 입력 항목
1. 서버 이름
2. Base URL
3. API Key
4. 자동 연결 옵션(있는 경우)

### 결과
생성 성공 시 해당 서버 화면으로 이동합니다.

## 6-D. 서버 편집 페이지
### 주요 기능
1. 서버 이름/URL/API Key 수정
2. 저장 후 복귀
3. 취소 시 이전 화면 복귀

### 사용 시점
1. API Key 교체
2. 서버 URL 변경
3. 서버 표시명 정리

---

## 6-E. 서버 관리 페이지 (Manage servers)
### 주요 기능
1. 등록 서버 검색
2. Import servers
3. Export servers
4. Add a server
5. 서버별 수정/삭제/옵션 접근

### 실무 팁
1. 팀 이동/PC 교체 시 Export로 백업
2. 신규 환경 세팅 시 Import로 복원

## 6-F. 서버 상세 페이지 (Shlink 내장 화면)
서버 상세는 Shlink Web Component 화면으로, 서버 운영 핵심 기능이 들어있습니다.

일반적으로 다음 기능을 사용합니다.
1. 단축 URL 목록 조회
2. 단축 URL 생성
3. URL 상세/수정
4. 태그, QR, 방문 통계 등 운영 기능

참고:
실제 메뉴 구성은 연결한 Shlink 서버 버전과 설정에 따라 일부 다를 수 있습니다.

## 6-G. Settings 페이지
### 주요 기능
1. UI 테마 및 표시 관련 설정
2. 목록 정렬/동작 기본값 설정
3. 사용자 환경설정 저장

### 권장 사용
1. 팀 공통 스크린샷/보고서가 있다면 테마를 통일
2. 목록 정렬 기본값을 업무 방식에 맞춰 고정

## 6-H. 사용자 관리 페이지 (관리자 전용)
### 탭 구성
1. 승인 대기
2. 활성
3. 비활성

### 주요 기능
1. 승인 대기 사용자 승인/거절
2. 활성 사용자 비활성화
3. 비활성 사용자 재활성화

## 6-I. UTM 기능 전체
UTM은 별도 업무 영역으로, 3개 페이지로 구성됩니다.

1. UTM 빌더
2. 템플릿 관리
3. 태그 관리

상단 UTM 관리 메뉴로 페이지 간 빠르게 전환할 수 있습니다.

### UTM 빌더
1. 기본 URL + UTM 값 입력
2. 생성 URL 실시간 확인
3. 복사
4. 단축링크 만들기 연결
5. 템플릿 적용
6. 템플릿 저장

### 템플릿 관리
1. 템플릿 생성
2. 템플릿 수정
3. 템플릿 삭제
4. 템플릿 구성 태그 확인

### 태그 관리
1. 카테고리별 태그 등록
2. 태그 수정
3. 태그 삭제
4. 설명 관리

---

## 7. UTM 고급 사용 가이드
### 7-1. 템플릿 저장 시 자동 태그 저장
UTM 빌더에서 템플릿 저장 시:
1. 템플릿 저장
2. 입력값 기반 태그 자동 저장

중복 처리:
1. 동일 카테고리+값은 중복 생성하지 않음
2. 없는 값만 추가

### 7-2. 권장 네이밍 규칙
1. 소문자 사용
2. 공백 대신 하이픈/언더스코어
3. 캠페인명에 기간/목적 포함

예시:
1. source: google, meta, newsletter
2. medium: cpc, social, email
3. campaign: 2026q2_brand_awareness

### 7-3. 실무 운영 루틴
1. 주 1회 태그 중복/오타 정리
2. 월 1회 미사용 템플릿 정리
3. 캠페인 종료 후 아카이브 규칙 적용

---

## 8. 권한 및 계정 운영 가이드
### 8-1. 역할 개요
1. member: 일반 운영 기능
2. admin: 사용자 관리 포함 전체 운영

### 8-2. 사용자 상태 개요
1. pending: 승인 대기
2. active: 활성(로그인 가능)
3. inactive: 비활성(로그인 제한)

### 8-3. 온보딩 권장 절차
1. 신규 가입
2. 관리자 승인
3. 기본 서버 등록/접속 테스트
4. UTM 템플릿 1개 생성 실습

---

## 9. 데이터/보안 운영 가이드
### 9-1. 서버 정보 관리
서버 URL/API Key는 민감 정보입니다.
공유 문서에 원문 키를 그대로 남기지 않는 것을 권장합니다.

### 9-2. Import/Export 사용 시 주의
1. Export 파일은 안전한 저장소에 보관
2. 외부 전송 시 암호화 채널 사용
3. 퇴사/권한 변경 시 키 회전 권장

### 9-3. 브라우저 로컬 데이터
서버 목록 등 일부 정보는 브라우저 로컬 저장소에 저장될 수 있습니다.
공용 PC 사용 후 로그아웃/브라우저 정리를 권장합니다.

## 10. 문제 해결 가이드
### 10-1. 로그인 안 됨
원인:
1. 승인 대기 계정
2. 비활성 계정
3. 이메일/비밀번호 불일치

해결:
1. 관리자 승인 상태 확인
2. 계정 상태 active 확인
3. 비밀번호 재확인

### 10-2. 서버 접속 실패
원인:
1. URL 오입력
2. API Key 오류
3. 서버 네트워크 제한

해결:
1. URL 끝 슬래시/프로토콜 확인
2. API Key 재발급 후 재등록
3. 서버 상태 점검

### 10-3. UTM URL 생성이 비정상
원인:
1. 기본 URL 형식 오류
2. 필수값 누락

해결:
1. https:// 포함 확인
2. source/medium/campaign 입력

### 10-4. 화면에서 메뉴가 안 보임
원인:
1. 권한 부족(예: 사용자 관리)
2. 현재 선택 서버 없음

해결:
1. 관리자 계정 여부 확인
2. 상단 Servers에서 서버 선택

## 12. 자주 묻는 질문
### Q1. 서버를 몇 개까지 등록할 수 있나요?
실무적으로 여러 개 등록해 운영할 수 있습니다.

### Q2. UTM 없이도 앱 사용이 가능한가요?
가능합니다. UTM은 선택 기능이며, 서버/단축 URL 운영만으로도 사용 가능합니다.

### Q3. 관리자 메뉴가 안 보입니다.
현재 계정 role이 admin이 아닐 가능성이 큽니다.

### Q4. 템플릿 저장 시 태그가 계속 늘어날까 걱정됩니다.
동일 값은 중복 생성하지 않으므로 무한 중복은 방지됩니다.

### Q5. 비전공자도 운영 가능한가요?
가능합니다. 이 문서의 빠른 시작과 페이지별 절차를 순서대로 따라하면 됩니다.
