## 1. UTM이란?
UTM은 링크에 붙는 추적 정보입니다.
어떤 경로로 유입되었는지 분석 도구에서 식별할 때 사용합니다.

주요 항목:
1. utm_source: 유입 출처
2. utm_medium: 유입 방식
3. utm_campaign: 캠페인 이름
4. utm_term: 검색어/세부 키워드
5. utm_content: 소재/버튼/배너 구분

예시:
`https://example.com/event?utm_source=google&utm_medium=cpc&utm_campaign=spring_sale`

---

## 2. 처음 시작하기
### 2-1. 준비물
1. 홍보할 기본 URL
2. 캠페인 규칙(예: source, medium 표준값)
3. 팀에서 합의한 네이밍 규칙

### 2-2. 꼭 알아둘 점
1. 일반적으로 source/medium/campaign은 필수로 입력합니다.
2. 값 표기를 통일하지 않으면 분석 데이터가 분산됩니다.
3. 템플릿을 저장하면 반복 작업 속도가 크게 빨라집니다.

---

## 3. 전체 화면 구조
UTM 기능은 다음 3개 페이지로 구성됩니다.

1. UTM 빌더
- 실제 추적 링크를 생성하는 메인 화면

2. 템플릿 관리
- 자주 쓰는 조합(템플릿) 생성/수정/삭제

3. 태그 관리
- 항목별 자주 쓰는 값(태그) 사전 관리

또한 UTM 페이지 안에는 상단 UTM 관리 메뉴(빌더/템플릿 관리/태그 관리)가 있어 화면을 빠르게 이동할 수 있습니다.

---

## 4. 빠른 시작: 3분 사용법
### Step 1. UTM 빌더로 이동
UTM 메뉴에서 빌더 화면으로 들어갑니다.

### Step 2. 기본 URL 입력
추적하고 싶은 원본 링크를 넣습니다.

### Step 3. 필수 UTM 값 입력
utm_source, utm_medium, utm_campaign을 입력합니다.

### Step 4. 결과 URL 확인 후 복사
자동 생성된 URL을 복사해 사용합니다.

### Step 5. 자주 쓸 조합이면 템플릿 저장
템플릿 이름을 입력하고 저장하면 다음부터 클릭 한 번으로 재사용할 수 있습니다.

---

## 5. 페이지별 상세 기능
## 5-A. UTM 빌더 페이지
### 5-A-1. 기본 URL 입력
- URL 입력칸에 주소를 넣으면 UTM 파라미터가 자동 조합됩니다.
- 이미 UTM 파라미터가 포함된 URL을 넣으면 각 입력칸에 자동 반영됩니다.

### 5-A-2. UTM 필드 입력
- source, medium, campaign, term, content를 각각 입력합니다.
- source/medium/campaign이 없으면 생성/복사/단축링크 흐름이 제한될 수 있습니다.

### 5-A-3. 자동완성(태그 기반 제안)
- 태그 관리에 저장된 값을 기준으로 입력 추천이 표시됩니다.
- 같은 값을 반복 입력할 때 오타를 줄이고 속도를 높일 수 있습니다.

### 5-A-4. 생성된 URL 영역
- 입력값을 조합한 최종 URL을 실시간으로 보여줍니다.
- 잘못된 URL 형식이면 정상 URL이 생성되지 않을 수 있습니다.

### 5-A-5. 복사 버튼
- 생성된 URL을 클립보드로 복사합니다.

### 5-A-6. 단축링크 만들기 버튼
- 서버 선택 상태에서 단축링크 생성 화면으로 값을 넘깁니다.

### 5-A-7. 템플릿 검색/적용
- 저장된 템플릿 목록에서 항목을 클릭하면 UTM 값이 즉시 채워집니다.
- 마우스 오버 시 강조되어 선택 대상을 쉽게 구분할 수 있습니다.

### 5-A-8. 현재 설정을 템플릿으로 저장
- 템플릿 이름과 설명을 입력 후 저장합니다.
- 저장 시 템플릿뿐 아니라 현재 값 기반 태그도 자동 반영됩니다.

---

## 5-B. 템플릿 관리 페이지
### 5-B-1. 템플릿 입력 폼
- 템플릿 이름, 설명, 각 UTM 값을 직접 입력해 새 템플릿 생성

### 5-B-2. 템플릿 수정
- 저장된 카드에서 수정 클릭
- 폼으로 값이 채워지고, 변경 후 수정 저장 가능

### 5-B-3. 템플릿 삭제
- 템플릿 카드에서 삭제 가능
- 삭제 전 확인 절차를 권장

### 5-B-4. 템플릿 구성 정보 확인
- 저장된 템플릿이 어떤 UTM 값으로 구성되는지 카드에서 확인 가능

### 5-B-5. 태그 설명 기반 입력 보조
- 입력 중 태그 설명이 함께 보여 선택 정확도를 높입니다.

---

## 5-C. 태그 관리 페이지
### 5-C-1. 태그 등록
- 카테고리(source/medium/campaign/term/content)
- 태그 값
- 설명(선택)

### 5-C-2. 태그 수정
- 목록에서 수정 선택 후 값/설명 변경

### 5-C-3. 태그 삭제
- 사용하지 않는 태그 정리

### 5-C-4. 카테고리별 태그 목록
- 카테고리별로 정리되어 한눈에 관리 가능

### 5-C-5. 실무 의미
- 태그 관리는 팀의 용어 사전 역할을 합니다.
- 빌더/템플릿 입력 품질에 직접 영향을 줍니다.

---

## 6. 템플릿 자동 태그 저장 기능
UTM 빌더에서 템플릿 저장을 누르면 다음이 함께 수행됩니다.

1. 템플릿 저장
2. 현재 입력된 UTM 값을 태그 목록에 자동 저장

중복 처리 방식:
1. 같은 카테고리 + 같은 값(대소문자 무시)이 이미 있으면 새로 만들지 않음
2. 없는 값만 추가됨

장점:
1. 사용자 입력 패턴이 자동으로 태그 사전에 축적됨
2. 다음 입력부터 자동완성 품질이 높아짐

주의:
1. 잘못 입력한 값도 태그로 누적될 수 있으므로 주기적으로 정리 필요

## 7. 실무 운영 가이드(구현 외 권장사항 포함)
아래 항목은 화면에 버튼이 없어도 팀 운영 품질을 위해 권장되는 규칙입니다.

### 7-1. 네이밍 규칙 표준화
1. 소문자 사용
2. 공백 대신 하이픈 또는 언더스코어 사용
3. 캠페인명 형식 통일

예시:
- campaign: `2026q2_brand_awareness`
- source: `google`, `meta`, `newsletter`
- medium: `cpc`, `social`, `email`

### 7-2. 템플릿 명명 규칙
템플릿 이름은 채널 + 목적 + 대상이 드러나게 작성

예시:
- `meta_retgt_cart_2026q2`
- `newsletter_weekly_product`

### 7-3. 콘텐츠(content) 규칙
동일 캠페인 내 소재 구분값으로 사용

예시:
- `hero_banner_a`
- `cta_footer_button`

### 7-4. 캠페인 수명 주기 관리
1. 시작 전: 태그/템플릿 준비
2. 운영 중: 신규 값 최소화
3. 종료 후: 사용 빈도 낮은 태그 정리

### 7-5. 품질 점검 루틴
주 1회 권장 점검:
1. 오타 태그 존재 여부
2. 중복 의미 값 존재 여부
3. 미사용 템플릿 정리
4. 캠페인 규칙 준수 여부

## 8. 오류/문제 해결 가이드
### 8-1. URL이 생성되지 않음
원인:
1. 기본 URL 형식이 잘못됨
2. 필수값(source/medium/campaign) 미입력

해결:
1. `https://` 포함 여부 확인
2. 필수 항목 채운 뒤 다시 확인

### 8-2. 템플릿이 보이지 않음
원인:
1. 저장되지 않았거나 삭제됨
2. 계정/사용자 기준 데이터 분리

해결:
1. 템플릿 관리 페이지에서 존재 확인
2. 로그인 계정 상태 확인

### 8-3. 자동완성 후보가 예상과 다름
원인:
1. 태그 누적 데이터에 오타/변형값 포함

해결:
1. 태그 관리에서 오타/중복 정리
2. 표준 용어 재적용

### 8-4. 값은 같은데 중복처럼 보임
원인:
1. 대소문자/공백 차이

해결:
1. 태그 입력 시 소문자 기준으로 통일
2. 앞뒤 공백 제거 후 저장

---

## 9. 팀 적용 체크리스트
아래 항목을 모두 만족하면 운영 안정성이 높습니다.

1. source/medium 표준 목록을 문서화했다.
2. campaign 명명 규칙이 있다.
3. 템플릿 이름 규칙이 있다.
4. 태그 정리 담당자 또는 주기가 정해져 있다.
5. 신규 담당자 온보딩 문서에 이 설명서를 포함했다.

## 10. 자주 묻는 질문
### Q1. UTM 값 하나에 여러 값을 넣을 수 있나요?
원칙적으로 각 항목은 하나의 의미를 갖는 값으로 관리하는 것을 권장합니다.

### Q2. 템플릿 저장 시 태그가 무조건 생성되나요?
없는 값만 생성됩니다. 이미 있는 값은 중복 생성되지 않습니다.

### Q3. term/content는 꼭 써야 하나요?
필수는 아니지만, 소재/키워드 분석을 하려면 사용하는 것이 좋습니다.

### Q4. 비전공자도 운영 가능한가요?
가능합니다. 이 문서의 빠른 시작과 페이지별 절차만 따라도 기본 운영이 가능합니다.
