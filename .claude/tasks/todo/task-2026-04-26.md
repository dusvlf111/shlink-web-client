## 적용 스킬 핵심 규칙 요약

### code-quality
1. 가독성 — 한 함수에 한 책임, 위→아래 자연스러운 흐름
2. 예측 가능성 — 이름·시그니처·반환값으로 동작 추정 가능
3. 응집도 — 함께 변경되는 코드를 가깝게 배치
4. 결합도 — 외부 패키지(ShlinkWebComponent)에 깊게 의존하지 않기

### vercel-react-best-practices (해당 작업에서 핵심)
- `rendering-hoist-jsx` — 정적 메뉴 정의 배열은 컴포넌트 외부로 호이스팅
- `rerender-functional-setstate` — 객체 상태 업데이트는 `(prev) => ({ ...prev, ...patch })`
- `rerender-dependencies` — `useMemo` 의존성 배열에 새 상태 누락 금지

### folder-structure
- UTM 전용 신규 컴포넌트 → `src/utm/`
- 다른 도메인이 재사용 가능한 레이아웃 → `src/common/`
