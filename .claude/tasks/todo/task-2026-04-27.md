## 전체 적용 스킬 핵심 규칙 요약

### folder-structure
- 기존 도메인 내부 수정 우선, 공용화는 필요가 명확할 때만 수행
- 도메인 간 직접 의존 최소화
- 관련 로직은 해당 도메인(share, utm, servers, common, i18n) 내부에 응집

### code-quality
- 가독성: 필수/선택, shortCode/fullUrl 등 의미가 다른 값을 네이밍으로 명확히 구분
- 예측 가능성: URL/검증/상태변경 로직은 함수로 분리해 동일 입력 동일 출력 보장
- 응집도: 기능별 변경을 해당 모듈에 모아 유지보수성 확보
- 결합도: UI가 저장소/API 세부사항에 직접 결합되지 않도록 훅/서비스로 캡슐화

### vercel-react-best-practices
- rendering-conditional-render: 상태별 렌더 분기를 명시적으로 작성
- rendering-hoist-jsx: 정적 메타/상수 JSX는 컴포넌트 외부로 호이스팅
- rerender-memo: 리스트/옵션 UI 등 재렌더 비용이 큰 영역은 memo와 stable props 검토
- client-swr-dedup / async-parallel: 데이터 조회 중복 호출 방지와 독립 비동기 병렬 처리 우선
