# PRD: 마케팅팀 후속 개선 (2026-04-27)

> 원본: `.claude/tasks/0427-1.md`
> 작성일: 2026-04-27

## 요구사항

### 1. 첫 페이지 웰컴 문구 + 서버 리스트 일관성

**현재 상태:**
- `src/common/Home.tsx` 가 `Welcome!` 라는 헤딩과 `useServers()` 의 redux 로컬 스토어 데이터를 이용해 서버 리스트를 보여줌
- 서버 등록(`CreateServer.tsx`)은 redux + redux-localstorage-simple 로 로컬 브라우저에만 저장
- 한편 `src/servers/useServerConfigs.ts` 는 PocketBase `server_configs` 콜렉션에서 서버 목록을 가져오는 별도 훅이 존재 (이미 `ServerInfoFloating` 에서만 사용)

**원하는 상태:**
- 홈 헤딩 `Welcome!` → `서버 리스트` 로 변경
- 서버 리스트 출처를 PocketBase `server_configs` 로 통합 → 모든 사용자/디바이스 간 일관 동작
- 신규 서버 등록도 PocketBase 에 저장 (로컬 redux 도 그대로 캐시 역할은 유지 가능)
- 예시 항목: `letscareer (l.letscareer.co.kr)`

### 2. 서버 정보 플로팅 버튼 삭제

- `src/common/ServerInfoFloating.tsx` (우측 하단 fixed) 가 더 이상 필요 없음 — 서버 리스트가 홈/사이드바에 통합됨
- App.tsx 의 `<ServerInfoFloating />` 호출 + 컴포넌트 파일 모두 제거

### 3. 사용자 관리 메뉴 배지

- 헤더의 `사용자 관리` 메뉴 우측 상단에 **승인 대기 (`status: 'pending'`) 사용자 수** 를 작은 카운트 배지로 표시
- 어드민 사용자에게만 보임 (현재도 이 메뉴는 admin 전용)
- 0 인 경우 배지 비표시

### 4. 단축링크 하위 메뉴 클릭 안 되는 버그 (긴급)

**증상:** 단축링크 메뉴 5개(대시보드/단축 링크 목록/단축 링크 만들기/태그 관리/도메인 관리)가 회색으로 보이고 클릭 불가

**원인:** `UnifiedSidebar.tsx` 가 `useParams<{ serverId }>()` 로만 활성 서버를 판단. 홈/UTM 페이지는 URL 에 serverId 가 없어 모든 단축링크 메뉴가 `requiresServer && !hasServer` → disabled 처리됨

**해결:** URL 에 serverId 가 없으면 redux 의 `useServers()` 에서 fallback 서버 선정 (autoConnect → 첫 번째 → null 순서). 그 서버 ID로 prefix 구성. 등록된 서버가 없을 때만 disabled.

### 5. 모바일 UI/UX 최적화

- 통합 사이드바가 현재 `max-md:hidden` 으로 모바일에서 보이지 않음 → 햄버거 토글로 대체
- 헤더 반응형 점검
- UTM 페이지 좌측 마진 모바일에서 0 으로 (이미 `md:pl-` 로 한정됐는지 확인)
- 페이지 본문 모바일 패딩 조정

## 우선순위

1. **Push 1 (긴급)**: 단축링크 사이드바 클릭 버그 — 운영 차단 이슈
2. **Push 2**: 홈 페이지 + 서버 PocketBase 통합 + 플로팅 버튼 삭제
3. **Push 3**: 사용자 관리 배지
4. **Push 4**: 모바일 최적화

## 비기능 요구사항

- 모든 신규/수정 라벨은 i18n 키 사용 (Push 1 으로 도입된 `useT()` 활용, ko/en locale 양쪽 추가)
- redux 로컬 캐시 → PocketBase 마이그레이션 시 로그인하지 않은 사용자에게도 로컬 fallback 유지 (회귀 방지)
