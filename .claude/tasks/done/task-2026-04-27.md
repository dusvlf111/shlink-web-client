## 적용 스킬 핵심 규칙 요약

### code-quality (4가지 기준)
1. 가독성 — 한 함수에 한 책임, 명확한 네이밍
2. 예측 가능성 — 함수 시그니처/반환값으로 동작 추정 가능
3. 응집도 — 함께 변경되는 코드를 가깝게 (admin 훅은 `src/admin/`, 서버 서비스는 `src/servers/services/`)
4. 결합도 — PocketBase 어댑터를 한 파일에 격리, redux 와 PB 양방향 동기화 단방향 흐름

### vercel-react-best-practices
- `rendering-hoist-jsx`: 정적 메뉴/배지 정의는 모듈 스코프
- `rendering-conditional-render`: `count > 0 && <Badge />` 단순 조건부
- `rerender-functional-setstate`: 객체 상태 업데이트는 `(prev) => ({ ...prev, ...patch })`
- `rerender-dependencies`: useMemo/useCallback 의존성 누락 금지

### folder-structure
- 신규 admin 훅 → `src/admin/`
- 신규 서버 서비스 → `src/servers/services/`
- 통합 UI/공용 컴포넌트 → `src/common/`
- i18n 메시지 → `src/i18n/locales/`
